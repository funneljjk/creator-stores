# Creator Hub Builder — 기획서 (PRD)

> 상태: v1 승인됨 · MVP 1차 구현 완료 (2026-06-23)
>
> **구현 완료**: 소셜 링크 자동발견(`src/discover.js`) · 블로그 RSS(`src/feeds.js`) ·
> 허브 홈(소셜바 + 유튜브 피드 + 인스타 카드 + 블로그 피드 + 커머스).
> 검증: @mkbhd → X·인스타·페북 자동발견 + 인스타 카드 렌더 / @jangproai → 영상·쇼츠 피드.
> 블로그 RSS 파서 = Medium·Atom 검증. **남음**: 인스타 OAuth(B), 스레드/틱톡, 빌더 UI에 수동 소셜 입력란.

## 1. 한 줄 정의
유튜브 링크 하나로, 크리에이터의 **모든 채널(유튜브·인스타·블로그)** 과 **강의·상품 판매**를
한 페이지에 모은 **올인원 허브 + 커머스 홈페이지**를 자동 생성·배포하는 SaaS.

## 2. 타겟
- **1순위**: 유튜버 · 인플루언서 (분석 시드가 유튜브)
- **2순위**: 블로거 · 스레더, 개인/쇼핑몰 판매자
- 공통 페르소나: 채널이 여러 곳에 흩어져 있고, 직접 사이트 만들 시간·기술이 없는 1인 크리에이터.

## 3. 문제
- 채널이 유튜브·인스타·블로그·스토어로 흩어짐 → 팬이 한곳에서 못 봄.
- 링크인바이오(Linktree)는 링크만 나열, 피드·판매 없음.
- 자사몰/홈페이지 직접 제작 = 시간·비용·기술 장벽.
- 수익화 수단(강의·상품·멤버십) 연결이 번거로움.

## 4. 핵심 가치
**"링크 하나 넣으면, 내 모든 채널 + 판매가 들어간 홈페이지가 5분 안에 완성·배포된다."**

## 5. 생성되는 허브 페이지 — 섹션 구조 (위 → 아래)
1. **배너/히어로** — 배너 이미지 + 프로필 + 태그라인 + 소셜 버튼바 + 주요 CTA
2. **소셜 허브** — 발견된 전 채널 아이콘 버튼 (유튜브·인스타·블로그·스레드·사이트)
3. **유튜브 피드** — 최근 영상 + 쇼츠 (임베드 재생) ✅이미 구현
4. **인스타** — 프로필 임베드 + 링크아웃 (간이 A방식)
5. **블로그 피드** — RSS 최신 글 카드 (네이버·티스토리·워드프레스·미디엄)
6. **강의 리스트** — 판매 ✅
7. **상품 리스트** — 판매 ✅
8. **푸터** — 채널 링크 · 문의

## 6. 기능 (MVP)
### 분석 / 자동 발견
- 유튜브 채널 분석 (영상·쇼츠·소개) ✅
- 채널 About에서 **소셜 링크 자동 발견** (인스타·블로그·사이트·트위터 등) 🆕
- 블로그 플랫폼 감지 → RSS URL 추론 🆕
- 추천엔진: 아키타입 분류 → 최적 솔루션/모듈 ✅
### 커머스
- 강의(vod)·상품·멤버십·코칭, 장바구니→결제→수익 대시보드 ✅
### 배포
- runmoa 키 검증 → 강의·상품·코칭 등록 ✅ (라이브 401 확인됨)
- 허브 페이지 = 정적 생성물

## 7. 데이터 흐름
```
유튜브 URL
  → analyze (yt-dlp)         : {채널, 영상, 쇼츠, About}
  → discover                 : About에서 소셜 링크 추출 + 블로그 RSS 추론
  → enrich                   : 블로그 RSS fetch, 인스타 임베드 메타
  → recommend                : 아키타입 → blueprint
  → generate                 : catalog + 허브 데이터 → web/js/data.js
  → preview (/store/)
  → deploy (runmoa)
```

## 8. 기술 현실 / 통합 등급
| 채널 | 자동 피드 | 방법 |
|---|---|---|
| 유튜브 | ✅ 완전 | yt-dlp / RSS |
| 블로그 | ✅ 완전 | RSS (네이버·티스토리·WP·미디엄) |
| 인스타 | ⚠️ 제한 | **MVP: 프로필 임베드 + 링크아웃 (A)**. 정식 OAuth(B)는 다음 |
| 스레드·틱톡 | ⚠️ 제한 | 링크아웃만 (피드 없음) |

## 9. MVP 범위 (이번 빌드)
**IN**
- 소셜 링크 자동 발견 (유튜브 About 스크레이프, 실패 시 수동 입력)
- 허브 홈 재설계: 배너 + 소셜바 + 유튜브피드 + 인스타임베드 + 블로그RSS + 강의/상품
- 블로그 RSS 연동 (네이버·티스토리·WP·미디엄)
- 인스타 = 임베드 / 링크아웃 (A)

**OUT (다음 단계)**
- 인스타 정식 OAuth 피드 (B)
- 스레드·틱톡 피드
- 커스텀 도메인, 방문자 통계, 문의폼, 뉴스레터, 후원
- runmoa 실 결제 연동

## 10. 아키텍처
```
server.js          /api/analyze(+discover) · /api/generate · /api/deploy
src/
  youtube.js       채널 분석
  discover.js  🆕  About → 소셜 링크 추출 + 블로그 RSS 추론
  feeds.js     🆕  블로그 RSS fetch + 정규화
  recommend.js     아키타입 → blueprint
  catalog.js       blueprint → 카탈로그 (+ hub 소셜/피드 데이터)
  mapper.js        runmoa 페이로드
  runmoa.js        API 클라이언트
web/               허브 SPA (섹션 모듈화: banner/social/yt/insta/blog/commerce)
web-builder/       위저드 (그대로)
```
- Node ≥18, npm 의존성 0개, yt-dlp.

## 11. 리스크 / 대응
- 인스타·스레드 자동피드 불가(ToS) → 임베드/링크로 우회 [수용]
- 채널 About에 소셜 링크 없을 수 있음 → 위저드에서 수동 입력 fallback
- 블로그 RSS 비표준 → 플랫폼별 패턴 + 실패 시 링크아웃
- 유튜브 About 링크가 yt-dlp 기본 출력에 없을 수 있음 → 채널 페이지 HTML 스크레이프 (빌드 시 검증 필요)

## 12. 미정 (나중에 결정)

- 허브 호스팅 위치 (정적 배포 타깃)
